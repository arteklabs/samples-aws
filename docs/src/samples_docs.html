AWS Profile Authentication#

About#

A profile is a collection of settings and credentials that authenticate a requester together with an AWS API. This collection can be found at ~/.aws/config and it contains the definitions of multiple AWS profiles:

[profile PROFILE]
sso_session = SSO-PORTAL
sso_account_id = 123456789ABC
sso_role_name = ROLE
region = REGION
output = json

[sso-session SSO-PORTAL]
sso_region = REGION
sso_start_url = https://SSO-PORTAL-URL.awsapps.com/start/
sso_registration_scopes = sso:account:access

Depending on the role that the authenticated profile can assume, temporary credentials are sent to the requester (sso/cache/*.json) enabling the requester to perform the authorized operations.

CLI Authentication#

Having installed the aws-cli-v2, configure your AWS profile with:

$ aws sso configure --profile $PROFILE
SSO session name (Recommended): SSO-PORTAL
SSO start URL [None]: https://SSO-PORTAL-URL.awsapps.com/start/
SSO region [None]: REGION
SSO registration scopes [sso:account:access]: sso:account:access
...
There are NUMBER_OF_ACCOUNTS_WITH_SSO_ACCESS AWS accounts available to you.
Using the account ID 123456789ABC
There are NUMBER_OF_ROLES roles available to you.
Using the role name "ROLE"
CLI default client Region [None]: eu-central-1
CLI default output format [None]: json

Notice that is has been created in ~/.aws/config:

[profile PROFILE]
sso_session = SSO-PORTAL
sso_account_id = 123456789ABC
sso_role_name = ROLE
region = REGION
output = json

[sso-session SSO-PORTAL]
sso_region = REGION
sso_start_url = https://SSO-PORTAL-URL.awsapps.com/start/
sso_registration_scopes = sso:account:access

Authorize your profile with ROLE by running:

$ aws sso login --profile $PROFILE

The authorization returns temporary credentials (sso/cache/*.json). The credentials have the following structure:

{
    "startUrl": "https://SSO-PORTAL-URL.awsapps.com/start/",
    "region": "REGION",
    "accessToken": "TMP_SSO_TOKEN",
    "expiresAt": "TMP_SSO_TOKEN_EXP_DATE"
}

SDK/CDK Authentication#

When using aws sso login on AWS CLI v2 as of July 27th, 2020, the credentials are stored so they will work with the CLI itself (v2) but donâ€™t work on the AWS SDKs and other tools that expect credentials to be readable from ~/.aws/credentials (v1). Until a solution is implemented in AWS CLI v2 is is necessary to update the ~/.aws/credentials file for AWS SSO users by updating/creating the corresponding profile section in ~/.aws/credentials with temporary role credentials. The ~/.aws/credentials file has the following structure:

[default]
aws_access_key = IAM_ACCESS_KEY_ID
aws_secret_access_key = IAM_ACCESS_KEY
aws_session_token = TMP_ACCESS_TOKEN

Extract the credentials stored at ~/.aws/config/sso/cache/*.json to ~/.aws/credentials by running:

npm run cpsso -- -p  PROFILE
